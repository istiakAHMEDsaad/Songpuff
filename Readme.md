# Requirements

## 1. Language and Localization
The application must support English (USA) and at least one additional language, such as German or Ukrainian.

When a language is selected, the song name, artist name, album name, and genre must be realistic and match that language naturally. No Lorem Ipsum or dummy text is allowed. These texts must not be hardcoded in the code; they must come dynamically from a third-party library or an external file.

## 2. Seed and Reproducibility
The user must be able to provide a 64-bit seed or click a **Random** button to generate a new seed.

Most importantly, the data must be generated based on the seed value and the page number. That means if a user enters a specific seed today, such as `12345`, the same songs, artists, or tunes generated on page 1 must be exactly the same one year later on any device, as long as the same seed is used.

## 3. Likes Per Song - Probabilistic Approach
The user must be able to input any value between 0 and 10, including fractional values such as `3.7`.

Behavior:
- If the user enters `0`, no song should have any likes.
- If the user enters `10`, every song should have exactly 10 likes.
- If the user enters `3.7`, likes must be distributed using mathematical probability so that the average likes across all songs on the page becomes 3.7. For example, some songs may have 3 likes and some may have 4 likes.

## 4. User Interface and Modes
### Toolbar
The controls for language, seed, and likes input must appear in a single line at the top of the screen as a toolbar.

### Dynamic Updates
There must be no submit or enter button. The data should update automatically as soon as the slider or seed changes.

### Two View Modes
#### Table View
- Pagination is required, with Next and Previous buttons.
- Clicking a row should expand it to show a large album cover, review, and play button.

#### Gallery View
- No pagination is allowed.
- Infinite scrolling must load new cards as the user scrolls down.

### Reset Policy
Whenever any toolbar parameter changes, the table view must return to page 1, and the gallery view must reset to the very top of the scroll position.

## 5. Song and Cover Generation
### Audio Playback
When the user clicks the Play button, the song or tune must play directly in the browser.

The music must be fully generated by code or algorithm, using music theory, chord progressions, and tempo mixing. It must also depend on the seed value.

### Cover Image
The album cover art must display the song name and artist name beautifully. The background must not be a simple gradient or plain lines; it should include an attractive pattern or artistic design.

## 6. Architecture Rule - The Golden Rule
### Server-Side Generation
No fake data should be stored in a database. The browser must not generate the data directly.

Whenever the browser requests a page, the server (backend) must instantly generate the data in memory according to the seed and return the response.
